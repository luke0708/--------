# 项目容器化交付标准 (Prompt for AI)

## 任务背景
我正在构建一个基于 Docker Compose 的微服务架构。
本机（Mac mini）负责开发，另一台机器（MacBook Air）负责统一部署运行。
你需要确保你生成的代码符合以下“容器化标准”。

## 必须包含的文件
在项目根目录下，必须生成以下两个文件：

### 1. `Dockerfile`
请根据项目语言（Python/Node.js/Go）生成生产级别的 Dockerfile。
- **基础镜像**：使用轻量级版本 (如 `python:3.11-slim` 或 `node:18-alpine`)。
- **工作目录**：统一为 `/app`。
- **依赖安装**：利用缓存机制（先复制依赖描述文件安装依赖，再复制源代码）。
- **端口暴露**：容器内部统一暴露端口（例如 `8000`），不要映射到宿主机（映射由外部 docker-compose 控制）。
- **启动命令**：使用稳定命令启动服务。

### 2. `.dockerignore`
必须包含此文件，排除不必要的文件以减小镜像体积：
- `.git`
- `__pycache__`
- `venv` / `node_modules`
- `.DS_Store`
- `.env` (密钥严禁打包进镜像)

## 代码规范
- **配置分离**：所有的敏感配置（API Key、数据库密码、Secret Key）必须通过 `os.getenv` 或 `process.env` 读取，**严禁硬编码**。
- **日志**：所有日志直接输出到标准输出 (stdout/stderr)，不要写入本地文件，以便 Docker 收集。
