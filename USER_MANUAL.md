# 📖 用户使用手册 (User Manual)

本手册旨在帮助最终用户理解如何通过本系统监控和控制其分布式项目。

---

## 💻 访问入口

| 服务 | 访问地址 | 功能描述 |
| :--- | :--- | :--- |
| **管理面板 (Infra-Dash)** | `http://localhost:39999` | 查看服务状态、管理容器启停。 |
| **日志中心 (Dozzle)** | `http://localhost:39998` | 实时滚动查看所有容器的运行日志。 |
| **新闻追踪器 (示例)** | `http://192.168.1.137:50003` | 已部署的具体业务系统首页。 |

## 🛠️ 日常操作流程

### 1. 检查总体健康状况
打开 **Infra-Dash** 首页，绿色表示服务正常，红色表示需要关注。右上角的刷新按钮可以手动强制更新状态。

### 2. 部署一个新功能
作为用户，你只需完成以下操作：
1. 在 **Mac mini** 告诉 AI：“完成这个功能并按规范打包。”
2. 获取生成的 `backend.zip` 文件并传输至 Mac Air。
3. 把文件放进主目录，告诉部署 AI：**“自动触发部署协议”**。

### 3. 查看实时日志
如果不确信系统是否正在抓取数据，访问 **Dozzle**，选择 `news-tracker` 容器，你可以看到每一次 AI 筛选和翻译的轨迹。

## ⚠️ 常见问题排查 (FAQ)

- **Q: 点了管理面板没反应？**
  - A: 确保 Docker Desktop 已在运行，且 `server/` 目录下的容器已启动。
- **Q: 新闻列表没刷新？**
  - A: 检查 `.env` 中的 API Key 是否过期，或在日志中心查看是否有 `401 Unauthorized` 报错。
- **Q: 端口被占用怎么办？**
  - A: 部署 AI 会自动处理冲突并去 `Port-Registry/` 分配新端口，你只需查看 AI 告知的新端口号即可。
